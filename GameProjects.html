Vertex shaders and GPU instancing in an unreleased game

In order to create this demo. I began by baking two animations from blender into textures: The egg creature running, and the book page turning (which was simply a physics simulation in blender). From there, I created two special shaders in Unity using URP and Unity shadergraph: One that animates a mesh using a texture and an extra set of uvs, and another that allows one to scrub through frames of a vertex animation via scripts. The result is a way of animating many meshes in a scene, while still allowing the use of GPU instancing to reduce draw calls. This technique also allows the use of pre-made physics-based animations in Unity without much difficulty. Note that many visual features of this game, such as the UI, and some special effects, have been removed to help showcase this feature.
